## ğŸ”§ **ğŸ” Step-by-Step: How STS Issues Temporary Credentials**

The process involves **two IAM components**:

1. âœ… The **caller (IAM user, role, federated identity)** must have **an identity-based policy** allowing them to call `sts:AssumeRole` to assume the target role.
2. âœ… The **target IAM role** must have a **trust policy** that explicitly allows the caller to assume it.

### 1ï¸âƒ£ **Caller Sends `sts:AssumeRole` Request**

The calling entity (an IAM user, Lambda, EC2 role, or external identity) sends a request to STS to assume a specific IAM role.

> this done via the `sts:AssumeRole` API call or `aws sts assume-role` CLI command.

### 2ï¸âƒ£ **STS Verifies Identity-Based Permissions on the Caller**

The **caller** (e.g., `arn:aws:iam::111111111111:user/John`) must have a policy that **grants permission to call `sts:AssumeRole` on the target role**:

âœ… **Callerâ€™s identity-based policy** (attached to user or calling role):

```json

{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": "sts:AssumeRole",
      "Resource": "arn:aws:iam::222222222222:role/AdminRole"
    }
    // ... other statements
  ]
}
```

> This does **not** give them access to what the role _can do_ â€” it only allows them to **assume it**.

### 3ï¸âƒ£ **STS Verifies the Trust Policy on the Target Role**

The **target role** must **trust the caller** â€” this is defined in the roleâ€™s **trust policy**.

âœ… **Roleâ€™s trust policy** (resource-based):

```json

{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Principal": {
        "AWS": "arn:aws:iam::111111111111:user/John"
```

> ğŸ“Œ This is where **the role says who is allowed to assume it**.  
> ğŸ“Œ If this doesnâ€™t match, the assume role request is denied **even if the caller has `sts:AssumeRole` allowed in their policy**.

### 4ï¸âƒ£ **STS Issues Temporary Security Credentials**

If both previous checks pass:

- âœ… The caller is **authorized to call `sts:AssumeRole`**
- âœ… The target role **trusts the caller**

Then **STS returns temporary credentials**:

```json

  "SecretAccessKey": "abc...",
  "SessionToken": "IQoJb...",
  "Expiration": "2025-06-04T13:00:00Z"

```

### 5ï¸âƒ£ **Caller Uses Temporary Credentials**

The caller can now interact with AWS services, but **only based on the permissions attached to the role they assumed** (i.e., the **permissions policy of the target role**).
